# sre-dojo

SREのための稽古場。本物のAWS環境を「作って、壊して、直して」を繰り返しながら、インフラ設計・障害対応・運用スキルを鍛える。

## これは何？

個人のSRE訓練環境。Terraformで本番に近いAWSインフラを構築し、意図的に障害を注入して、検知・調査・復旧の一連の流れを練習する。

LocalStackなどのシミュレーションでは得られない、本物のAWSでの手触りを重視。カオスエンジニアリングだけでなく、インフラ設計のPoC・負荷テスト・新技術の実験場としても使う。

## アーキテクチャ

### 対象サービス

最小のGo APIサーバー。訓練の「対象物」であり、主役はインフラ側。

```
GET /health          → 200 OK
GET /api/info        → サービス情報（uptime, version等）
GET /simulate/slow   → 意図的に遅延して応答
GET /simulate/error  → 意図的に500を返す
```

### インフラ構成

```
┌─────────────────────────────────────────┐
│                  VPC                     │
│  ┌──────────┐    ┌───────────────────┐  │
│  │   ALB    │───▶│   ECS Fargate     │  │
│  │          │    │   (Go API)        │  │
│  └──────────┘    └───────────────────┘  │
│                         │               │
│                  ┌──────┴──────┐        │
│                  │ CloudWatch  │        │
│                  │ Logs/Metrics│        │
│                  └─────────────┘        │
└─────────────────────────────────────────┘
```

Terraformで管理。使うときだけ立ち上げてコストを最小化する設計。
